# 개선 사항 추적

이 문서는 코드 리뷰로 도출한 개선 항목과 처리 상태를 한곳에 모읍니다.

**갱신 방법**: "업데이트해줘" 라고 요청하면 코드를 다시 검토해서
아래 표의 상태를 갱신하고, 문서 끝 **변경 이력** 에
`[개선 사항_YYYY-MM-DD]` 항목을 새로 추가합니다.

**상태 표기**: ✅ 완료 · 🔄 진행 중 · ⬜ 미착수
**중요도**: 🔴 높음 · 🟡 중간 · 🟢 낮음

## 📚 책 DB 설계

별도 주제로 다룬 항목입니다.

| 단계 | 내용 | 상태 |
|---|---|---|
| 1 | 정규화 컬럼(`norm_title`/`norm_author`) + 유니크 인덱스, 기존 중복 병합 | ✅ |
| 2 | 장르 룩업 테이블(`genres`) + `books.genre_id` + 등록 화면 `<select>` 전환 | ✅ |
| 3 | 관리자 도서 정리 화면 (오타 수정·중복 병합·`verified` 플래그) | ⬜ |
| 4 | ISBN 기반 외부 API 도입 (Edge Function 프록시로 키 보호) | ⬜ |

3단계는 사용자가 10명을 넘어가면 필요합니다. 4단계는 그 이후 판단해도 늦지 않습니다.
