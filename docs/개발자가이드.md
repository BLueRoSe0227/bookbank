# 개발자 가이드

코드를 고치기 전에 읽어주세요. 이 프로젝트가 왜 이런 모양인지와,
지키지 않으면 조용히 깨지는 규칙들을 정리했습니다.

---

## 전체 구조

```
브라우저 (정적 파일)                    Supabase
┌────────────────────┐              ┌──────────────────────┐
│ index.html         │              │ Auth (로그인)         │
│ css/*.css          │  ──HTTPS──▶  │ Postgres + RLS       │
│ js/*.js            │              │ RPC 함수 (포인트 로직) │
│ sw.js (PWA)        │              │ pg_cron (연체·알림)   │
└────────────────────┘              └──────────────────────┘
   Cloudflare Pages
```

**서버 코드가 없습니다.** 백엔드 로직은 전부 Postgres 함수 안에 있고,
브라우저는 Supabase SDK로 그 함수를 호출할 뿐입니다.

### 왜 빌드 도구가 없나

의도적입니다. `npm run build` 없이 파일을 그대로 올리면 배포가 끝나고,
브라우저에서 소스를 그대로 읽을 수 있어 유지보수 부담이 낮습니다.

대신 다음을 감수합니다.
- ES 모듈 대신 IIFE + 전역 객체 (스크립트 로드 순서에 의존)
- 번들링·최소화 없음 (파일 11개를 개별 요청)
- 타입 검사 없음

## 모듈 작성 규약

각 파일은 IIFE로 감싸고 전역 상수 하나만 노출합니다.

```js
const Books = (() => {
  let _inited = false;              // 내부 상태는 _ 로 시작
  const _private = () => { ... };   // 내부 함수도 _ 로 시작

  const init = async () => { ... };

  return { init, submitLoan };      // 밖에서 쓸 것만 반환
})();
```

- `_` 접두사가 붙은 것은 그 파일 밖에서 쓰지 않습니다
- 모듈 간 호출은 `App.showToast(...)` 처럼 전역 객체를 통합니다
- 새 모듈을 추가하면 `.eslintrc.json` 의 `globals` 와 `varsIgnorePattern` 에도 이름을 넣어야 합니다

## ⚠️ XSS 방어 — 가장 중요한 규칙

사용자가 입력한 값을 화면에 넣을 때는 **반드시** 태그드 템플릿 `App.h` 를 씁니다.

```js
// ✅ 올바름 — ${} 안이 자동으로 이스케이프됩니다
el.innerHTML = App.h`<h3>${book.title}</h3>`;

// ❌ 위험 — 책 제목에 <script> 가 있으면 그대로 실행됩니다
el.innerHTML = `<h3>${book.title}</h3>`;
```

의도적으로 HTML을 넣어야 할 때만 `App.raw()` 로 감쌉니다.

```js
// 내부에서 만든 값이라 안전한 경우
App.h`<span class="badge badge-${App.raw(cls)}">${label}</span>`
```

> **`App.raw()` 에는 사용자 입력을 절대 넣지 마세요.**
> 넣어야 한다면 `App.escapeHtml()` 을 먼저 통과시킵니다.

`textContent` 로 넣는 경우는 이스케이프가 필요 없습니다.

### 인라인 이벤트 핸들러 금지

`onclick="..."` 을 쓰면 CSP를 걸 수 없습니다. `index.html` 에는 `onclick` 이 0개이며,
이 상태를 유지해야 합니다. 대신 `data-action` 속성 + `events.js` 의 액션 맵을 씁니다.

```html
<button data-action="submit-room" type="button">만들기</button>
```

```js
const actions = {
  'submit-room': () => Bookshelf.submitRoom(),
};
```

동적으로 그린 요소는 `addEventListener` 로 직접 연결합니다.

| `App.rules()` / `App.genres()` | 포인트 규칙 / 장르 목록 (캐시됨) |
| `App.errMsg(error, fallback)` | DB 오류를 사용자 문장으로 변환 |
| `App.today()` / `App.addDays(n)` / `App.toISODate(d)` | **날짜는 반드시 이것들을 쓰세요** |

### ⚠️ 날짜 처리

**`toISOString()` 을 날짜 문자열 만드는 데 쓰지 마세요.** UTC 기준이라
한국 시간 자정 근처에서 하루가 어긋납니다. `App.toISODate()` / `App.today()` 를 쓰세요.

```js
// ❌ KST 00:30 에 실행하면 어제 날짜가 나옵니다
new Date().toISOString().slice(0, 10)

// ✅
App.today()
```

### 모달 열고 닫기

`classList.add('active')` 를 직접 만지지 마세요. 포커스 복원이 깨집니다.

```js
// ❌
modal.classList.remove('active');

// ✅
App.closeModal('bsRoomModal');
```

## 보안 모델

이 프로젝트의 핵심 설계입니다. 이해하지 않고 고치면 구멍이 납니다.

### 3중 방어

1. **GRANT/REVOKE** — 문 자체를 잠급니다
2. **RLS 정책** — 문 안에서 자기 것만 보이게 합니다
3. **SECURITY DEFINER 함수** — 돈이 오가는 로직은 함수 안에서만

### 클라이언트가 직접 쓸 수 있는 것

**`profiles.nickname` 하나뿐입니다.**

```sql
revoke update on profiles from authenticated;
grant  update (nickname) on profiles to authenticated;
```

이 두 줄이 없으면 사용자가 자기 잔액과 권한을 마음대로 바꿀 수 있습니다.

`loans`, `transactions`, `overdue_logs`, `books`, `genres` 는 쓰기가 전부 차단되어 있고,
반드시 RPC 함수를 통해야 합니다.

### 새 기능을 추가할 때

포인트가 오가거나 상태를 바꾸는 기능이라면:

1. **`SECURITY DEFINER` 함수**로 만들고 `set search_path = public` 을 붙입니다
2. 함수 첫 줄에서 **권한을 확인**합니다 (`is_member()` / `is_admin()`)
3. `grant execute` 로 호출 권한을 줍니다
4. 내부 전용 함수라면 `revoke execute ... from authenticated, anon, public`

```sql
create or replace function my_feature(...)
returns json
language plpgsql
security definer set search_path = public   -- ← 필수
as $$
begin
  if not is_member() then raise exception '승인된 회원만 이용할 수 있습니다.'; end if;
  ...
end;
$$;

grant execute on function my_feature to authenticated;
```

> `adjust_points()` 는 잔액을 직접 바꾸는 함수라 사용자 호출이 차단되어 있습니다.
> 절대 `grant execute` 하지 마세요.

### 동시성

`adjust_points()` 는 `select ... for update` 로 행을 잠급니다.
잔액을 다루는 코드는 반드시 이 함수를 거쳐야 합니다. 직접 `update profiles set balance` 하지 마세요.

## 책 DB 설계

같은 책을 하나의 행으로 모으기 위해 **정규화 컬럼**을 씁니다.

```sql
norm_title  text generated always as (book_norm(title))  stored
norm_author text generated always as (book_norm(author)) stored
```

`book_norm()` 은 공백과 구두점을 지우고 소문자로 만듭니다.
`"해리 포터 (1권)"` → `"해리포터1권"`

유니크 인덱스가 `(norm_title, norm_author)` 에 걸려 있어
띄어쓰기만 다른 책은 자동으로 합쳐집니다.

> ⚠️ `book_norm()` 의 정의를 바꾸면 이미 저장된 generated 컬럼 값과 어긋납니다.
> 고쳐야 한다면 컬럼을 drop → 재생성해서 전체를 다시 계산하세요.

장르는 `genres` 룩업 테이블에서 관리하며, 등록 화면의 `<select>` 도 이 표를 읽어 그립니다.

## 마이그레이션 작성 규칙

`supabase/migrations/NNN_설명.sql` 형식입니다.

1. **`begin;` / `commit;` 으로 감쌉니다** — 중간 실패 시 전체 취소
2. **여러 번 실행해도 안전하게** — `if not exists`, `on conflict do nothing`, `create or replace`
3. **`schema.sql` 에도 같은 내용을 반영합니다** — 신규 설치 경로가 어긋나지 않도록
4. 파일 상단에 **왜 필요한지** 주석으로 남깁니다

> `create or replace function` 은 **인자 목록이 같으면** 기존 `grant` 를 유지합니다.
> 인자를 바꿔야 한다면 `drop function` 후 재생성하고 `grant execute` 를 다시 해야 합니다.

## 서비스 워커

앱 셸(HTML·JS·CSS)은 **네트워크 우선**, 나머지 정적 자원은 **캐시 우선**입니다.

> ⚠️ **`index.html` 이나 `js/` 를 고쳤다면 `sw.js` 의 `CACHE` 상수를 올리세요.**
> 안 올리면 기존 사용자가 옛 화면에 고착될 수 있습니다.
> CI가 이걸 잊으면 경고를 남깁니다.

```js
const CACHE = 'library-bank-v4';   // ← 배포할 때마다 올림
```

`CHANGELOG.md` 의 버전과 맞춰두면 나중에 추적하기 쉽습니다.

## CSP (콘텐츠 보안 정책)

`_headers` (Cloudflare Pages 용) 와 `index.html` 의 `<meta>` 두 곳에 있습니다.
**둘 다 고쳐야 합니다.**

외부 스크립트나 API를 추가하면 CSP도 함께 열어야 하지만,
**가능하면 열지 말고 self-host 하거나 Edge Function 프록시를 쓰세요.**

### 외부 스크립트를 쓸 때

버전을 패치까지 고정하고 SRI 해시를 답니다.

```bash
curl -s https://cdn.jsdelivr.net/npm/패키지@버전/파일.js \
  | openssl dgst -sha384 -binary | openssl base64 -A
```

```html
<script src="...@2.110.7/dist/umd/supabase.js"
        integrity="sha384-..."
        crossorigin="anonymous"></script>
```

> API 키가 필요한 외부 API는 **절대 클라이언트에서 직접 호출하지 마세요.**
> `js/config.js` 에 넣으면 즉시 노출됩니다. Edge Function 프록시를 쓰세요.

## 로컬 개발

```bash
npx serve .          # 또는 python -m http.server 8000
```

`file://` 로 열면 서비스 워커와 CSP가 동작하지 않습니다. 반드시 HTTP 로 띄우세요.

### 린트

```bash
npm install
npm run lint         # eslint js sw.js --max-warnings 0
```

**경고 0건을 유지합니다.** CI가 경고 하나에도 실패합니다.

### 서비스 워커 때문에 변경이 안 보일 때

개발자 도구 → Application → Service Workers → **Update on reload** 를 켜거나,
Unregister 후 새로고침하세요.

## 자주 하는 실수

| 실수 | 결과 |
|---|---|
| `App.h` 없이 `innerHTML` 에 사용자 입력 | XSS 취약점 |

| RPC 없이 테이블 직접 쓰기 시도 | 권한 오류 (의도된 동작) |
