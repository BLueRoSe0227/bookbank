# 관리자 가이드

서비스를 운영하는 사람을 위한 안내입니다.
설치·배포는 [배포안내.md](../배포안내.md) 를, 코드 수정은 [개발자가이드.md](개발자가이드.md) 를 보세요.

---

## 관리자가 하는 일

| 주기 | 할 일 |
|---|---|
| 수시 | 가입 승인 (신규 가입자가 방치되지 않게) |
| 주 1회 | 통계 탭에서 연체 처리가 정상인지 확인 |
| 주 1회 | 앱에 한 번 접속 (7일 방치 시 DB가 잠깁니다) |
| 필요 시 | 포인트 조정, 잘못 등록된 도서 정보 정정 |

---

## 첫 관리자 만들기

가입만으로는 관리자가 되지 않습니다. 최초 1회는 SQL로 직접 승격해야 합니다.

1. 먼저 앱에서 **일반 가입**을 마칩니다
2. Supabase 대시보드 → **SQL Editor**
3. 아래를 실행합니다

```sql
update profiles
   set role = 'admin', approved_at = now()
 where nickname = '내닉네임';
```

4. 앱을 새로고침하면 하단에 **관리** 탭(🛡️)이 나타납니다

> 승격만으로는 가입 보너스 500원이 지급되지 않습니다. 필요하면
> 관리 → 회원 → 포인트 조정으로 직접 넣어주세요.

### 관리자 추가하기

두 번째 관리자부터도 같은 SQL을 씁니다. 앱 화면에는 권한 변경 기능이 없습니다.
(실수로 권한이 바뀌는 것을 막기 위해 의도적으로 뺐습니다.)

> ⚠️ **마지막 관리자는 탈퇴할 수 없습니다.** 서비스가 운영 불능이 되는 것을 막는 장치입니다.

---

## 관리 화면

하단 **관리** 탭에 4개의 하위 탭이 있습니다.

### 승인 대기

가입 신청자 목록입니다. **관리 탭 아이콘과 이 탭에 빨간 배지로 대기 인원이 표시됩니다.**

- **승인**: 회원으로 전환하고 가입 보너스 500원을 지급합니다
- **거절**: 가입 신청을 **삭제**합니다 (확인 창이 뜹니다)

> 거절해도 같은 이메일로 다시 가입할 수 있습니다.

### 회원

승인된 회원 목록입니다. 닉네임, 잔액, 가입일이 보이고 관리자에게는 배지가 붙습니다.

**포인트 조정** 버튼으로 잔액을 직접 늘리거나 줄일 수 있습니다.

- 음수를 입력하면 차감입니다 (예: `-50`)
- **사유 입력이 필수**입니다. 이용자의 통장 내역에 "관리자 조정: (사유)" 로 남습니다
- **1회 조정 한도는 ±10,000원**입니다
- 차감 결과가 마이너스가 되면 실패합니다

### 대출 현황

전체 회원의 대출 중·연체 중인 책을 반납 예정일 순으로 보여줍니다 (최대 50건).

### 통계

운영 판단에 쓰는 화면입니다.

| 지표 | 의미 |
|---|---|
| **이번 달 완독** | 북극성 지표. 이 서비스의 성공은 대출이 아니라 반납입니다 |
| 이번 달 완독자 | 실제로 책을 끝낸 사람 수 |
| 완독률 | 지금까지 만들어진 대출 중 반납으로 끝난 비율 |
| 대출 중 / 연체 중 / 회원 수 | 현재 상태 |

그 아래에 **연체 처리 상태**와 **최근 14일 활동**이 표시됩니다.

## 연체 처리 감시 ⚠️

**가장 자주 문제가 생기는 부분입니다.**

연체료는 `pg_cron` 이 매일 자동으로 부과합니다.
그런데 Supabase 무료 플랜은 **7일간 접속이 없으면 프로젝트를 일시정지**시킵니다.
정지된 동안에는 cron 도 멈추므로 연체료가 걷히지 않습니다.

### 확인 방법

관리 → **통계** 탭의 "연체 처리 상태" 를 봅니다.

- 🟦 **"정상입니다"** — 밀린 것이 없습니다
- 🟨 **"아직 부과되지 않은 연체가 N건 있습니다"** — 자동 처리가 멈췄을 가능성

### 복구 방법

**지금 연체 처리 실행** 버튼을 누르면 됩니다.

내부적으로 놓친 날짜를 **모두 소급해서** 부과하므로, 며칠 멈췄더라도
한 번만 누르면 복구됩니다. 이미 부과된 날짜는 중복 부과되지 않습니다.

### 예방

- **주 1회 이상 앱에 접속**하면 프로젝트가 잠들지 않습니다
- Supabase 대시보드 → Database → Extensions 에서 **pg_cron 이 켜져 있는지** 확인하세요

### 자동 실행 시각

| 작업 | 실행 시각 (한국 시간) | 하는 일 |
|---|---|---|
| `overdue-daily` | 매일 00:05 | 연체료 부과, 상태를 `overdue` 로 변경 |
| `reminders-daily` | 매일 09:00 | 반납 하루 전·연체 알림 생성 |

알림은 앱 홈 화면 상단 배너로만 표시됩니다. **메일이나 푸시는 가지 않습니다.**

## 도서 정보 정정

이용자가 오타를 냈거나 잘못된 정보를 등록한 경우입니다.

도서 정보는 여러 사람이 함께 쓰는 공용 데이터라 앱에서는 수정할 수 없습니다.
현재는 SQL로 직접 고쳐야 합니다.

```sql
-- 특정 책 찾기
select id, title, author, publisher, genre, total_pages
  from books
 where title ilike '%검색어%';

-- 수정
update books
   set title = '올바른 제목', author = '올바른 저자'
 where id = 123;
```

### 중복된 책 합치기

띄어쓰기만 다른 같은 책이 두 개로 갈라진 경우입니다.
(003 마이그레이션 이후에는 자동으로 방지되지만, 저자 표기가 다르면 여전히 갈라질 수 있습니다.)

```sql
-- 남길 책 id 를 keep, 없앨 책 id 를 drop 이라 할 때
update loans           set book_id = <keep> where book_id = <drop>;
update bookshelf_books set book_id = <keep> where book_id = <drop>;
delete from books where id = <drop>;
```

> 순서가 중요합니다. 참조를 먼저 옮기지 않고 삭제하면 외래키 오류가 납니다.

## 포인트 규칙 바꾸기

규칙은 `app_config()` 함수에 하드코딩되어 있습니다.
바꾸려면 SQL Editor 에서 함수를 다시 정의합니다.

```sql
create or replace function app_config(key text)
returns int language sql immutable as $$
  select case key
    when 'join_bonus'    then 500   -- 가입 보너스
    when 'loan_deposit'  then 100   -- 대출 보증금
    when 'return_bonus'  then 15    -- 정시 반납 보너스
    when 'overdue_fee'   then 10    -- 연체료 (하루)
    when 'room_fee'      then 200   -- 서재 방 추가
    when 'extend_fee'    then 50    -- 연장 수수료
    when 'extend_days'   then 7     -- 연장 일수
    when 'extend_max'    then 2     -- 연장 최대 횟수
    else 0 end;
$$;
```

> ⚠️ 이미 진행 중인 대출의 보증금은 대출 시점의 값으로 `loans.deposit` 에
> 저장되어 있어, 규칙을 바꿔도 **반납 시 원래 맡긴 금액이 그대로 환급됩니다.**

> ⚠️ 이용자 화면은 규칙을 한 번 읽어 캐시합니다. 바꾼 뒤에는 새로고침해야 반영됩니다.

## 장르 목록 바꾸기

장르는 `genres` 테이블에서 관리합니다. 한 줄 추가하면 앱의 선택 목록에 바로 나타납니다.

```sql
insert into genres (name, sort) values ('여행', 95);
```

`sort` 는 표시 순서입니다. 기존 값 사이에 넣고 싶으면 중간 숫자를 주세요.

## 마이그레이션 적용

DB 구조를 바꾸는 SQL 파일입니다. `supabase/migrations/` 에 번호순으로 있습니다.

### 적용 방법

1. Supabase 대시보드 → **SQL Editor** → **New query**
2. 파일 내용을 **전체** 복사해 붙여넣기
3. **Run**
4. 오류 없이 끝나면 완료

### 적용 순서

**반드시 번호순으로** 실행하세요. 건너뛰면 실패합니다.

| 파일 | 내용 |
|---|---|
| `002_manual_book_entry.sql` | 도서 검색 API 제거, 알림·이벤트 로그 |
| `003_book_normalize_genres.sql` | 책 제목 정규화, 장르 룩업 테이블 |
| `004_admin_stats.sql` | 관리자 통계, 연체 처리 수동 실행 |

> 처음부터 새로 설치하는 경우에는 `supabase/schema.sql` 하나만 실행하면 됩니다.
> 마이그레이션 파일은 **이미 운영 중인 DB**를 업데이트할 때만 씁니다.

### 적용 전 주의

- 마이그레이션은 전체가 하나의 트랜잭션이라 **중간에 실패하면 아무것도 반영되지 않습니다**
- 여러 번 실행해도 안전하게 작성되어 있습니다
- 다만 `003` 의 중복 도서 병합은 **행을 삭제하므로 되돌릴 수 없습니다**

### 적용 확인

```sql
-- 003 적용 여부
select count(*) from genres;                    -- 13이면 적용됨
select column_name from information_schema.columns
 where table_name = 'books' and column_name = 'norm_title';

-- 004 적용 여부
select proname from pg_proc where proname = 'admin_stats';
```

## 백업 ⚠️

**무료 플랜에는 자동 백업이 없습니다.**

정기적으로 직접 받아두시길 강하게 권합니다.

- Supabase 대시보드 → Database → **Backups** 에서 수동 다운로드
- 또는 SQL Editor 에서 주요 테이블을 조회해 CSV 로 내보내기

회원 수가 적더라도 **대출 기록과 거래 내역은 복구가 불가능**합니다.

## 문제 해결

| 증상 | 원인과 조치 |
|---|---|
| "서버에 연결할 수 없어요" | 프로젝트가 7일 방치로 정지됨. Supabase 대시보드에서 **Restore** |
| 연체료가 안 걷힘 | 통계 탭 → "지금 연체 처리 실행". pg_cron 확장이 켜져 있는지도 확인 |
| 승인했는데 이용자가 못 들어감 | 이용자에게 새로고침 요청. 그래도 안 되면 `profiles.role` 확인 |
| 통계 탭이 안내 문구만 표시 | `004_admin_stats.sql` 미적용 |
| "서버 업데이트가 필요합니다" | 마이그레이션 미적용. 번호순으로 실행 |
| 이용자가 옛날 화면을 봄 | `sw.js` 의 `CACHE` 버전을 올리고 재배포했는지 확인 |
| 관리 탭이 안 보임 | `profiles.role` 이 `'admin'` 인지 확인 후 새로고침 |

## 알아둘 제약

- **푸시 알림·메일 알림이 없습니다.** 승인 대기도 관리자가 직접 확인해야 합니다
- **가입 승인이 수동입니다.** 관리자가 없으면 신규 유입이 멈춥니다
- **무료 플랜 한도**: 개인·소규모 모임 용도로는 넉넉하지만, 7일 방치 정지에 유의하세요
- **도서 정보 수정 화면이 없습니다.** 현재는 SQL로만 가능합니다
