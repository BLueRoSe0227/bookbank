## 전체 그림

```
   [ Cloudflare Pages ]          [ Supabase ]
    화면(HTML/CSS/JS)   ←──→   로그인·DB·연체자동처리
         무료                        무료
```

**왜 두 군데인가요?**
Supabase는 데이터베이스와 로그인을 담당하지만 **HTML 화면은 호스팅하지 않습니다.**
그래서 화면만 Cloudflare Pages에 올립니다. 폴더를 드래그하면 끝이라 5분이면 됩니다.

> 📖 **책 정보는 직접 입력합니다.** 예전엔 네이버 도서 API로 검색했지만,
> 외부 API에 기대면 그쪽이 문을 닫는 순간 앱이 멈춥니다. 지금은 서버 코드가
> 아예 없어서 API 키도, Edge Function도, Supabase CLI도 필요 없습니다.

# 2단계. 데이터베이스 만들기 (10분)

여기가 이 앱의 심장입니다. 테이블·보안규칙·포인트 로직·연체 자동처리가 **한 파일에** 들어있습니다.

### 2-1. pg_cron 켜기 (연체 자동 처리용)

1. 왼쪽 메뉴 **Database** → **Extensions**
2. 검색창에 `pg_cron` 입력
3. 토글을 켜기 → **Enable extension**

> 이게 연체료를 매일 자동으로 계산해줍니다. PHP판에서 외부 스케줄러가 하던 일입니다.

### 2-2. 스키마 실행

1. 왼쪽 메뉴 **SQL Editor** → **New query**
2. 받으신 zip에서 `supabase/schema.sql`을 메모장으로 엽니다
3. **전체 복사** (Ctrl+A → Ctrl+C)
4. SQL Editor에 붙여넣고 → 오른쪽 아래 **Run** (또는 Ctrl+Enter)
5. `Success. No rows returned` 가 나오면 성공

### 2-3. 확인

왼쪽 **Table Editor** 를 열어 테이블 **7개**가 보이는지 확인하세요:

```
profiles · books · loans · transactions
overdue_logs · bookshelf_rooms · bookshelf_books
```

<details>
<summary>❓ 빨간 에러가 났다면</summary>

- **`extension "pg_cron" is not available`** → 2-1을 건너뛴 겁니다. Extensions에서 켜고 다시 실행하세요.
- **`already exists`** → 이미 한 번 실행한 겁니다. 처음부터 다시 하려면 SQL Editor에서
  `drop schema public cascade; create schema public;` 실행 후 다시 붙여넣으세요.
  ⚠️ 데이터가 다 지워지니 처음 설치할 때만 쓰세요.
</details>

# 3단계. 접속 정보 넣기 (5분)

1. Supabase 대시보드 → 왼쪽 아래 **Settings**(톱니) → **API**
2. 두 값을 복사:
   - **Project URL** — `https://abcdefg.supabase.co`
   - **Project API keys** 의 **anon** / **public** 키 (아주 긴 문자열)
3. zip의 `js/config.js` 를 메모장으로 열고 **위 두 줄만** 바꿉니다:

```javascript
const SUPABASE_URL      = 'https://abcdefg.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOi...(아주 긴 문자열)';
```

4. 저장 (UTF-8)

> ### ❓ anon 키를 공개해도 되나요?
> **네, 괜찮습니다.** 이 키는 애초에 브라우저에 노출되도록 설계된 "공개용" 키입니다.
> 실제 보안은 DB의 RLS 정책이 담당합니다 — 로그인해도 남의 데이터는 DB가 막습니다.
> (제가 실제로 잔액 조작·권한 상승 등 6가지 공격을 시도해 전부 차단되는 걸 확인했습니다.)
>
> ### 🚨 단, **service_role** 키는 절대 넣지 마세요.
> 그건 모든 걸 뚫는 만능 키입니다. `config.js`에는 **anon** 키만 넣으세요.

# 알아둘 점 (제품 범위)

- **가입은 관리자가 손으로 승인합니다** (P6). 가족·소모임 같은 소규모용 설계입니다.
  많은 사람에게 열 계획이면 초대 링크 방식 등을 따로 고민해야 합니다.
- **한국어·개인용 앱입니다** (P10). 다국어나 여러 도서관 공유 기능은 넣지 않았습니다.
  버그가 아니라 의도된 범위입니다.
- **책 정보는 직접 입력합니다.** 도서명을 2글자 이상 치면 이미 등록된 책이
  자동완성으로 떠서, 같은 책을 재사용할 수 있습니다. (중복·오타 줄이기)
- **알림** (P3): 반납 하루 전·연체 시 앱 홈 상단에 배너로 알려줍니다.
  DB의 `generate_reminders()` 가 매일 만듭니다. 이메일까지 원하면
  `supabase/functions/send-reminders/` 를 배포하세요(선택). 없어도 앱 배너는 뜹니다.
- **이벤트 로그** (P8): 대출/반납/연장이 `events` 표에 익명 집계로 남습니다.
  조회는 관리자만 가능(RLS). 나중에 "얼마나 반납하나" 같은 개선 근거로 씁니다.

> 🔄 **이미 예전 버전으로 배포해 둔 경우**: `supabase/schema.sql` 을 다시 실행하면
> 데이터가 지워집니다. 대신 **`supabase/migrations/002_manual_book_entry.sql`** 을
> SQL Editor 에서 실행하세요. 데이터를 지키면서 이번 변경분만 적용합니다.

# 문제 해결

| 증상 | 원인과 해결 |
|---|---|
| 화면이 하얗게만 나옴 | `js/config.js`의 URL/키를 안 넣었습니다. F12 → Console 확인 |
| **"승인 대기 중"에서 안 넘어감** | 정상입니다. 관리자가 승인해야 합니다 (5단계) |
| 로그인이 안 됨 | Supabase → Authentication → Users 에서 계정이 있는지 확인 |
| 비번 재설정 메일의 링크가 엉뚱한 곳으로 | 4-1의 Site URL 설정을 안 한 겁니다 |
| **permission denied** 에러 | schema.sql 을 끝까지 실행 안 한 겁니다. 다시 전체 실행 |
| 대출 신청이 **함수를 못 찾는다**는 에러 | 구버전 DB입니다. `supabase/migrations/002_manual_book_entry.sql` 을 SQL Editor에서 실행하세요 |
| 화면을 고쳤는데 반영이 안 됨 | `sw.js` 의 `CACHE` 번호를 안 올린 겁니다. 서비스 워커가 옛 화면을 캐시하고 있습니다 |
| 갑자기 전부 안 됨 | 7일 방치로 잠긴 겁니다. Supabase 대시보드에서 **Restore** |

**로그 보는 법**: Supabase → **Logs** → `Postgres Logs`.
브라우저 문제는 **F12 → Console** 을 보세요.

# 포인트 규칙 (기본값)

| 항목 | 금액 | 설명 |
|---|---|---|
| 가입 승인 | **+500원** | 동시 5권까지 빌릴 수 있는 돈 |
| 대출 보증금 | **−100원** | 반납하면 **그대로 돌려받습니다** |
| 정시 반납 | **+15원** | 보증금과 별도 |
| 연체료 | **−10원/일** | 잔액 0에서 멈춤 |
| 대출 연장 | **−50원** | 7일 연장, 최대 2회, 환급 없음 |
| 서재 방 추가 | **−200원** | 첫 방은 무료 |

**핵심 아이디어: 잔액 = 대출 여력.**
빌리면 보증금이 묶여서 대출 여력이 줄고, 반납하면 돌아옵니다.
연체하면 여력 자체가 깎이니 실질적인 페널티가 됩니다.

바꾸고 싶으면 SQL Editor에서 `app_config` 함수의 숫자만 고치면 됩니다:

```sql
create or replace function app_config(key text)
returns int language sql immutable as $$
  select case key
    when 'join_bonus'    then 500   -- ← 이 숫자들
    when 'loan_deposit'  then 100
    when 'return_bonus'  then 15
    when 'overdue_fee'   then 10
    when 'room_fee'      then 200
    when 'extend_fee'    then 50
    when 'extend_days'   then 7
    when 'extend_max'    then 2
    else 0 end;
$$;
```
