# 독서 통장 (bookbank)

읽은 만큼 쌓이는 독서 기록 PWA. 책을 빌릴 때 포인트를 보증금으로 맡기고,
제때 반납하면 돌려받으며 보너스가 붙습니다. 완독한 책은 3D 서재에 꽂아 모읍니다.

정적 파일 + Supabase 조합이라 서버를 직접 운영하지 않습니다.

## 주요 기능

| 화면 | 하는 일 |
|---|---|
| 홈 | 잔액·대출 현황 요약, 반납 임박 안내 |
| 책 등록 | 도서 정보 직접 입력 + 이미 등록된 책 자동완성, 대출 신청 |
| 내 대출 | 대출 중/반납 목록, 반납(별점·메모), 연장 |
| 독서 통장 | 포인트 입출금 내역 |
| 포트폴리오 | 완독 통계(장르·작가·출판사), 업적 8종 |
| 나의 서재 | 방을 만들고 완독한 책을 책장에 꽂기 |
| 설정 | 닉네임·비밀번호 변경, 데이터 내보내기, 탈퇴 |
| 관리자 | 가입 승인, 회원 관리, 포인트 조정, 대출 현황, 통계 |

## 기술 구성

- **프런트**: 순수 HTML/CSS/JS (빌드 도구 없음). 모듈은 IIFE 패턴으로 `js/` 에 분리
- **백엔드**: Supabase (Postgres + Auth + RLS + Edge Functions + pg_cron)
- **호스팅**: Cloudflare Pages (정적)
- **PWA**: `manifest.json` + `sw.js` (앱 셸은 네트워크 우선, 정적 자원은 캐시 우선)

포인트가 오가는 로직은 전부 Postgres 함수(`SECURITY DEFINER`)에 있습니다.
클라이언트는 `profiles.nickname` 외에 어떤 테이블도 직접 쓸 수 없습니다
(`supabase/schema.sql` 하단 권한 블록 참고).

## 로컬에서 실행하기

빌드가 없으므로 정적 서버만 있으면 됩니다.

```bash
npx serve .
# 또는
python -m http.server 8000
```

`file://` 로 직접 열면 서비스 워커와 CSP가 동작하지 않으니 반드시 HTTP 로 띄우세요.

`js/config.js` 의 `SUPABASE_URL` / `SUPABASE_ANON_KEY` 는 자신의 프로젝트 값으로 바꿉니다.
anon 키는 공개되어도 되는 키입니다. 실제 접근 통제는 RLS가 담당합니다.
**`service_role` 키는 절대 여기에 두지 마세요.**

## Supabase 설정 순서

1. 새 프로젝트 생성
2. SQL Editor 에서 `supabase/schema.sql` 전체 실행
3. 이미 구버전으로 배포한 DB라면 `supabase/migrations/` 를 번호순으로 실행
4. Database → Extensions 에서 **pg_cron** 켜기 (연체료 자동 부과에 필요)
5. 첫 가입자를 관리자로 승격:
   ```sql
   update profiles set role = 'admin', approved_at = now()
    where nickname = '내닉네임';
   ```

자세한 배포 절차는 [배포안내.md](배포안내.md) 를 참고하세요.

## 성공 지표

이 프로젝트의 북극성 지표는 **월간 완독 권수**입니다.
대출이 아니라 반납이 성공입니다 — 빌리기만 하고 안 읽으면 의미가 없습니다.

보조 지표는 완독률(대출 대비 반납 비율)과 연체율입니다.
관리자 → 통계 탭에서 확인할 수 있습니다.

## 릴리스 전 확인 목록

배포 전에 아래를 직접 눌러봅니다. 자동화된 테스트가 아직 없기 때문입니다.

- [ ] 가입 → 관리자 승인 → 로그인
- [ ] 책 등록 → 대출 신청 (보증금 차감 확인)
- [ ] 반납 (별점 선택 → 보증금 환급 + 보너스 확인)
- [ ] 연장 (수수료 차감 확인)
- [ ] 서재 방 만들기 → **책 꽂기** → 책등 클릭 → 빼기
- [ ] 포트폴리오 통계·업적 표시
- [ ] 관리자 통계 탭, 연체 처리 수동 실행
- [ ] 다크 모드 전환
- [ ] 키보드만으로 모달 열기·닫기 (Tab 이 모달 밖으로 나가지 않는지)
- [ ] `sw.js` 의 `CACHE` 버전을 올렸는지
